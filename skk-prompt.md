# my-ime SKK辞書生成プロンプト

Perplexity、Claude、ChatGPTなどにそのまま貼って、`my-ime` 用のSKK辞書候補を作るためのプロンプトです。

下の `{TARGET}` を、取り込みたい分野・文章・URL・単語リストに置き換えて使います。出力されたSKK行は、人間が確認してから `data/*.skk` に追加してください。

```text
あなたは日本語IME用の小さな技術辞書を作る補助者です。
以下の入力から、SKK 形式の辞書候補を生成してください。

対象:
{TARGET}
2015年から現在までの人名、主な出来事、webサービス、技術用語、略語、IT、Web、AI、LLM、自然言語処理、機械学習、データサイエンス分野、カルチャー、ユースカルチャー、サブカルチャー、ゲーム、アニメ、漫画、映画、音楽、文学、社会現象など。

目的:
- ローマ字入力から、日本語・カタカナ・英字混じりの専門用語を素早く確定できるようにする。
- 一般的すぎる語ではなく、対象分野で実際に何度も入力しそうな語を優先する。
- 変換候補は短く、IME辞書として安全に使える表記にする。

辞書形式:
- 1行1エントリ。
- 形式は `key /候補/`。
- 複数候補が自然な場合だけ `key /候補1/候補2/`。
- `key` はASCII小文字のローマ字、英字略語、または短いハイフン入りローマ字にする。
- `key` には空白を入れない。
- `key` は原則としてユーザーが実際に打つ読み・略語にする。
- 候補は1つの語または短い定型句にする。
- 1行の末尾に説明文やコメントを付けない。

ローマ字キーの方針:
- 日本語読みは訓令式とヘボン式の揺れがある場合、実際に打ちやすいものを優先する。
- 長音を含むカタカナ語は、このプロジェクトでは `de-ta /データ/` のようにハイフンを使ってよい。
- 英語由来の技術語は、実際に英字で打つ形も使ってよい。例: `server /サーバー/`、`llm /LLM/`。
- 助詞付きでよく使うものは、必要に応じて次の派生を追加する。
  - `wo` -> `を`
  - `de` -> `で`
  - `ni` -> `に`
  - `no` -> `の`
  - `ga` -> `が`
  - `ha` -> `は`
- 自然な場合だけ、次の派生も追加する。
  - `suru` -> `する`
  - `ka` -> `化`
  - `teki` -> `的`
  - `gata` -> `型`
  - `kan` -> `間`

品質基準:
- 既存のSKK辞書やGPL辞書から行をコピーしない。
- Webで調べた場合も、既存辞書の丸写しではなく、一般的な用語理解から新しく作った候補にする。
- 表記が曖昧な語は、最も標準的な候補を1つ目に置く。
- 固有名詞、製品名、略語は、大小文字や公式表記を尊重する。
- 読みが不明な語、実際に使うか怪しい語、差別的・攻撃的・危険な語は除外する。
- 重複行を出さない。
- 多すぎる候補を避け、重要なものから最大120行までに絞る。

出力:
- まず `;;; source: <対象の短い説明>` というコメント行を1行だけ出す。
- その後にSKK行だけを出す。
- Markdownの箇条書き、表、説明文、コードフェンスは使わない。
- 最後に余計な解説を付けない。

出力例:
;;; source: local example
de-ta /データ/
de-tawo /データを/
server /サーバー/
serverwo /サーバーを/
llm /LLM/
llmde /LLMで/
kanji /漢字/感じ/
henkan /変換/返還/
```

## 確認メモ

- `data/*.skk` は worker 起動時に自動で読み込まれます。
- 上流SKK辞書からコピーした行は、ライセンス確認と通知文書の更新なしに入れないでください。
- 出力を追加したら、重複や表記ゆれを軽く見てから `make test` で確認します。
